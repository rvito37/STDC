# STDC Export Tool

PowerShell скрипт для экспорта данных стандартных затрат (Standard Cost) из DBF-файлов в Excel.

## Что делает скрипт (`export_xlsx.ps1`)

Скрипт читает базу данных производственного учёта электронных компонентов (AVX) и собирает данные о стандартных затратах по каждой продуктовой линейке в один Excel файл.

### Источники данных (DBF файлы)

| Файл | Описание |
|------|----------|
| `C_WIP.DBF` | Индекс продуктов (WIP = Work In Progress). Содержит список продуктовых линеек (PLINE), ссылку на файл данных (WFILE) и описание продукта |
| `C_PROC.DBF` | Справочник процессов. Содержит PROC_ID → PCPROCNO (код процесса в системе учёта) |
| `Cxx.DBF` (C02, C51, C113...) | Таблицы стандартных затрат по процессам для конкретного продукта. ~500 файлов |

### Связи между таблицами

```
C_WIP.DBF                    Cxx.DBF                     C_PROC.DBF
+---------+--------+         +----------+-----------+     +----------+----------+
| PLINE   | WFILE  |         | PROC_ID  | STDC_DLAB |     | PROC_IDL | PCPROCNOL|
+---------+--------+         +----------+-----------+     +----------+----------+
| AF1     | C02    |-------->| 160.0    | 0.250     |---->| 160.0    | 4010     |
| AP1     | C113   |         | 170.0    | 0.170     |     | 170.0    | 4180     |
| AG1     | C191   |         | 173.0    | 1.800     |     | 173.0    | 4820     |
+---------+--------+         +----------+-----------+     +----------+----------+
   WFILE = имя файла            JOIN по PROC_ID
```

### Алгоритм работы

1. **Читает C_PROC.DBF** — строит словарь `PROC_ID → PCPROCNO`
2. **Читает C_WIP.DBF** — находит все уникальные PLINE (продуктовые линейки) и соответствующие файлы данных
3. **Для каждой PLINE** открывает указанный в WFILE файл (напр. C02.DBF) и:
   - Фильтрует записи: `PROC_TYPE ≠ "A"` (убирает вспомогательные процессы)
   - Фильтрует записи: хотя бы одно из полей STDC_DLAB/OVHD/MATL/DEPR/UPKP ≠ 0 (убирает пустые строки)
   - Подтягивает PCPROCNO из справочника C_PROC по PROC_ID
   - Сортирует по PCPROCNO
4. **Генерирует Excel** (SC_Export.xlsx) — по одному листу на каждую PLINE

### Выходные колонки

| Колонка | Описание |
|---------|----------|
| PCPROCNO | Код процесса из C_PROC (для сортировки и идентификации) |
| PROC_ID | ID процесса в таблице затрат |
| STDC_DLAB | Стандартная стоимость прямого труда (Direct Labor) |
| STDC_OVHD | Накладные расходы (Overhead) |
| STDC_MATL | Материалы (Materials) |
| STDC_DEPR | Амортизация (Depreciation) |
| STDC_UPKP | Обслуживание оборудования (Upkeep) |
| STDC_FCTR | Коэффициент стоимости (yield factor) |
| PROC_FCTR | Коэффициент процесса |

### Фильтры

- `PROC_TYPE ≠ "A"` — исключает auxiliary (вспомогательные) процессы
- Все STDC стоимостные поля = 0 — исключает записи без данных о затратах

## Запуск

```powershell
powershell -ExecutionPolicy Bypass -File export_xlsx.ps1
```

Результат: `SC_Export.xlsx` в той же папке.

### Требования

- Windows с PowerShell 5.1+
- DBF файлы (C_WIP, C_PROC, Cxx) в той же папке
- Не требует установки дополнительных модулей (генерирует XLSX через System.IO.Compression)

## Технические детали

- DBF файлы читаются напрямую через BinaryReader (формат dBASE III, version=3)
- Кодировка данных: CP866 (OEM Russian)
- XLSX генерируется как ZIP-архив с OpenXML внутри (без COM/Excel/внешних библиотек)
- Имена полей в DBF содержат мусорные байты после основного имени — скрипт их чистит regex `[^\x20-\x7E]`
